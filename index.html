therefore ∴

A snappy, native text editor. Every keystroke is zero-latency — and it is not an Electron app.

init.lua
-- therefore is configured in Lua. Saving applies it live.return {  theme       = "eink",          -- ink on warm paper  font        = "JetBrains Mono",  font_size   = 15,  line_numbers = true,  keys = {    toggle_git      = "ctrl+shift+g",    toggle_terminal = "ctrl+`",  },}

What it is

therefore is a text editor drawn on the GPU with winit, wgpu and glyphon — text and every panel, icon and graph line. Editing is local and synchronous, so the cursor never waits. The window idles at 0% CPU and wakes on input.

Anything that could block — git, SSH, the shell — runs on a background thread and reports back, so it never holds up a keystroke.

Features

Native and fast

No web view, no runtime. A portable build is a single executable — everything therefore needs is compiled in.

A real git window

A second OS window with the full commit graph, staging, diffs, branches, merging and stashes. Park it on another monitor.

Embedded terminal

A real PTY in a panel at the bottom of the editor, themed to match. Full ANSI color, drawn with the theme's palette.

Remote files over SFTP

Open user@host:/path, edit locally, save back over SSH. Nothing installed on the server, and editing stays as fast as a local file.

Incremental syntax highlighting

tree-sitter for sixteen languages. A keystroke reparses the edit, not the file, and only the rows on screen are queried.

Configured in Lua

Themes, fonts, sizes and keybindings in init.lua. Saving it applies immediately — no restart.

Build & run

Requires a Rust toolchain and a C compiler.

cargo run # scratch buffer
cargo run -- path/to/file.rs # a local file
cargo run -- user@host:/etc/hosts # remote, over SFTP
